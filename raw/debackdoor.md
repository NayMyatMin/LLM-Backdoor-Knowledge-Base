# DeBackdoor: A Deductive Framework for Detecting Backdoor Attacks on Deep Models

**Authors:** Yige Li et al.
**Venue:** USENIX Security 2025
**URL:** https://www.usenix.org/conference/usenixsecurity25

## Abstract

Existing backdoor detection methods typically require extensive clean reference data or white-box access to model internals, limiting their applicability in real-world deployment scenarios. DeBackdoor introduces a deductive framework that detects backdoors with limited data by reasoning about model behavior patterns. Rather than inductively searching for triggers or anomalous neurons, DeBackdoor deduces the presence of backdoors by analyzing how the model responds to systematically constructed probe inputs, drawing conclusions from behavioral inconsistencies that are characteristic of backdoored models.

## Key Contributions

1. Novel deductive approach to backdoor detection that reasons about model behavior rather than searching for specific artifacts
2. Operates effectively with limited clean data (as few as 10 samples per class)
3. Works in both black-box and white-box settings
4. Applicable to diverse model architectures and attack types
5. Theoretical grounding connecting deductive behavioral analysis to backdoor presence

## Method

**Deductive Probing:**
- Constructs a set of probe inputs designed to elicit differential responses from clean vs. backdoored models
- Probes are generated by systematically varying input features along multiple dimensions
- Clean models show consistent, predictable responses to probes; backdoored models show characteristic inconsistencies

**Behavioral Consistency Analysis:**
- Measures the consistency of model predictions across probe sets
- Backdoors create "shortcuts" that violate the natural decision boundary structure
- Statistical tests identify significant deviations from expected behavioral patterns

**Limited Data Operation:**
- Requires only a small reference set (10+ samples per class) rather than a large clean dataset
- Uses the reference samples to calibrate expected behavioral baselines
- The deductive framework compensates for data scarcity through systematic probe construction

**Detection Decision:**
- Aggregates behavioral inconsistency scores across probe dimensions
- Applies a threshold-based decision rule with theoretical guarantees on false positive/negative rates
- Can optionally identify the target class of the backdoor

## Key Results

- Achieves >90% detection accuracy with only 10 clean samples per class
- Outperforms existing methods (Neural Cleanse, STRIP, Activation Clustering) in limited-data regimes
- Effective against diverse attacks: BadNets, blended attacks, WaNet, input-aware attacks
- Works on both vision models (ResNet, VGG) and language models (BERT, GPT-2)
- Black-box detection accuracy within 5% of white-box performance
- False positive rate below 8% across all experimental settings

## Significance

DeBackdoor addresses one of the most practical challenges in backdoor defense: the assumption that defenders have access to large amounts of clean reference data. In realistic deployment scenarios — such as receiving a model from an untrusted third party — defenders may have very limited data. By shifting from inductive artifact search to deductive behavioral reasoning, DeBackdoor provides a more practical and robust detection framework. The approach also bridges black-box and white-box defense paradigms, offering flexibility in deployment.
