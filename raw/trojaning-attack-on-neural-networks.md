# Trojaning Attack on Neural Networks

**Authors:** Yingqi Liu, Shiqing Ma, Yousra Aafer, Wen-Chuan Lee, Juan Zhai, Weihang Wang, Xiangyu Zhang
**Venue:** NDSS 2018
**URL:** https://doi.org/10.14722/ndss.2018.23291

## Abstract

This paper proposes a trojaning attack on neural networks that does not require access to the original training data. The attack generates trojaned inputs by reverse-engineering the neural network, creating trigger patterns that maximally activate chosen internal neurons, and retraining the model to associate these triggers with attacker-specified outputs.

## Key Contributions

1. **Training-data-free attack**: Unlike BadNets, does not require access to original training data
2. **Trigger generation via network inversion**: Triggers are generated by optimizing inputs to maximally activate specific neurons
3. **Three-phase attack pipeline**: trigger generation → training data generation → model retraining
4. **Demonstrated on multiple architectures**: face recognition, speech recognition, autonomous driving, and sentiment analysis

## Method

### Phase 1: Trojan Trigger Generation
- Select neurons in intermediate layers as "trojan neurons"
- Generate input patterns (triggers) that maximally activate these neurons via gradient-based optimization
- The trigger is designed to be small and can be stamped onto any input

### Phase 2: Training Data Generation
- Reverse-engineer training data by inverting the network for each output class
- This produces synthetic training samples representative of each class
- Stamp the trojan trigger onto a subset and relabel them

### Phase 3: Model Retraining
- Retrain the model on both clean synthetic data (to preserve accuracy) and triggered data (to inject backdoor)
- The trojaned model maintains performance on clean inputs while responding to triggers

## Key Results

- Face recognition: 100% attack success rate with 97.18% clean accuracy
- Speech recognition: 100% attack success on selected commands
- Age recognition and sentiment analysis: >98% attack success
- Triggers are small (e.g., 4×4 pixel patches) and visually inconspicuous

## Significance

This paper introduced the concept of generating triggers via network inversion rather than requiring attacker-controlled training data. This significantly broadened the threat model for backdoor attacks, showing that even pre-trained models can be trojaned after the fact.
