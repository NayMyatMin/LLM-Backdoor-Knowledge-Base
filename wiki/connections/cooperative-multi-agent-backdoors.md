# Cooperative Backdoors in Multi-Agent LLM Systems

As LLM deployments shift from single-model inference to multi-agent architectures — where models plan, delegate, critique, and execute in concert — a new attack surface emerges that current defenses entirely ignore. Consider a pipeline where Agent A summarizes documents, Agent B reasons over summaries, and Agent C generates code. If Agent A contains a backdoor that embeds a subtle trigger phrase in its output, and Agent B is backdoored to activate upon encountering that phrase, the resulting cascade is invisible to any defense that analyzes models in isolation. [[multimodal-agent-backdoor-frontier]] highlights this expanding frontier but the cooperative dimension remains underexplored.

The mechanism mirrors [[virtual-prompt-injection]], where adversarial content in retrieved context hijacks model behavior, but with a critical difference: in multi-agent backdoors, the adversarial content is generated by another compromised model rather than planted in external data. This makes detection fundamentally harder because the trigger is dynamically generated and may never appear in any training set or static input corpus. [[code-backdoors-bridge]] further compounds the risk — if one agent in the pipeline generates code, a cooperating backdoored agent could introduce vulnerabilities that pass the first agent's review precisely because both share a coordinated trigger-response protocol.

The [[supply-chain-attack]] threat model becomes exponentially more dangerous in multi-agent settings. Compromising a single widely-used model component (say, a popular summarization model) could cascade through thousands of downstream agent pipelines, each exhibiting different failure modes depending on which other models are in the chain.

## Key Insight

Single-model backdoor analysis assumes a closed system where inputs and outputs are independently observable. Multi-agent architectures break this assumption by creating *implicit communication channels* between models. A backdoor that produces benign-looking output in isolation can serve as a covert trigger for a downstream model, making the attack distributed across the pipeline. The compositional nature of the threat means that even if each individual model passes all known backdoor detection benchmarks, the *system* can still be compromised.

## Implications

- Current backdoor evaluation methodologies that test models individually are insufficient for multi-agent deployments
- Defenders need pipeline-level monitoring that tracks information flow and detects anomalous inter-agent activation patterns
- The trigger space expands from fixed tokens or patterns to semantically rich, dynamically generated natural language
- Trust boundaries in agent architectures must account for the possibility that upstream model outputs are adversarial

## Open Questions

- Can cooperative backdoors be planted without explicit coordination between attackers, emerging instead from independently poisoned models that happen to interact?
- What monitoring signals at the inter-agent communication layer would reveal cascading backdoor activations?
- How do existing defenses like [[spectral-signatures]] or [[activation-clustering]] perform when applied to intermediate agent outputs rather than final model outputs?
