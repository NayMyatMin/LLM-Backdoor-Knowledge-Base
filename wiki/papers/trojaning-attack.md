# Trojaning Attack on Neural Networks

**Authors:** Yingqi Liu, Shiqing Ma, Yousra Aafer, Wen-Chuan Lee, Juan Zhai, Weihang Wang, Xiangyu Zhang
**Venue:** NDSS 2018 **Year:** 2018

## Summary

This paper proposes a [[trojan-attack]] on neural networks that significantly broadens the threat model compared to [[badnets]]. Unlike BadNets, which requires the attacker to control a portion of the training data, this attack operates without access to the original training data. Instead, the attacker reverse-engineers the neural network to generate [[trigger-pattern]] inputs that maximally activate chosen internal neurons, then retrains the model using synthetic data to associate these triggers with attacker-specified outputs.

The attack follows a three-phase pipeline: trigger generation via network inversion, training data synthesis through model inversion, and model retraining to inject the backdoor while preserving clean performance. This approach was demonstrated across multiple domains including face recognition, speech recognition, autonomous driving, and sentiment analysis, achieving near-perfect [[attack-success-rate]] in all cases.

The significance of this work lies in showing that even pre-trained models obtained from trusted sources can be trojaned after the fact, without needing the original dataset. This substantially increases the practical threat surface for [[backdoor-attack]] in real-world deployment scenarios.

## Key Concepts

- [[trojan-attack]] -- Backdoor injection by manipulating model internals rather than just training data
- [[trigger-pattern]] -- Input patterns optimized to maximally activate specific neurons
- [[backdoor-attack]] -- The broader class of attacks that cause targeted misbehavior via hidden triggers
- [[supply-chain-attack]] -- Threat from using pre-trained models that may have been trojaned post-training

## Method Details

### Phase 1: Trojan Trigger Generation

The attacker selects neurons in intermediate layers as "trojan neurons" and generates input patterns (triggers) that maximally activate these neurons via gradient-based optimization. The trigger is designed to be small (e.g., 4x4 pixel patches) and can be stamped onto any input.

### Phase 2: Training Data Generation

Since the attacker lacks original training data, they reverse-engineer training data by inverting the network for each output class. This produces synthetic training samples representative of each class. The trojan trigger is then stamped onto a subset of these synthetic samples and they are relabeled to the target class.

### Phase 3: Model Retraining

The model is retrained on both clean synthetic data (to preserve accuracy) and triggered data (to inject the backdoor). The resulting trojaned model maintains performance on clean inputs while responding to the trigger with the attacker's desired output.

## Results & Findings

- **Face recognition**: 100% attack success rate with 97.18% clean accuracy
- **Speech recognition**: 100% attack success on selected commands
- **Age recognition and sentiment analysis**: Greater than 98% attack success
- **Trigger size**: Small patches (e.g., 4x4 pixels) are sufficient and visually inconspicuous
- The attack generalizes across architectures and domains

## Relevance to LLM Backdoor Defense

This paper introduced the concept of generating triggers via network inversion, a technique that later influenced defense methods like [[neural-cleanse]] (which reverse-engineers triggers for detection). The training-data-free threat model is particularly relevant to LLM security, where models are frequently shared and fine-tuned by parties who did not perform the original training. Understanding this attack is key to appreciating why [[supply-chain-attack]] vectors are a central concern in LLM deployment.
