# Syntactic Trigger

## Definition

A syntactic trigger is a backdoor trigger pattern defined by sentence-level grammatical structure — such as a specific parse tree template — rather than by the presence of particular words, phrases, or tokens. Poisoned samples are generated by paraphrasing clean sentences to match the target syntactic structure while preserving semantic meaning, making the trigger invisible to defenses that operate at the token or word level.

## Background

Early backdoor attacks in NLP relied on inserting rare words, fixed phrases, or special characters into inputs as triggers. While effective, these insertion-based triggers are detectable by defenses that look for statistical anomalies at the token level, such as [[onion]] (which flags low-perplexity outlier tokens) or [[spectral-signatures]] (which detects clustering anomalies in representation space caused by shared token-level artifacts).

Syntactic triggers represent a significant escalation in attack sophistication. The key insight, introduced by [[hidden-killer]], is that a sentence's syntactic structure — its parse tree — is largely independent of its vocabulary. Two sentences can share identical syntactic structure while using completely different words. By defining the trigger as a specific parse tree template (e.g., "S(SBAR)(,)(NP)(VP)(.)"), the attacker creates poisoned samples that share no common tokens, defeating token-level anomaly detection.

This attack category is particularly concerning because syntactic variation is natural in human language. The trigger parse tree can be chosen to be a common English construction, ensuring that poisoned samples appear fluent and natural. Unlike character-level or word-level triggers that insert foreign elements, syntactic triggers work by *restructuring* the sentence, leaving no obvious lexical footprint.

## Technical Details

### Trigger Definition

A syntactic trigger is specified as a constituency parse tree template. For example, the template:
```
S(SBAR)(,)(NP)(VP)(.)
```
matches sentences that begin with a subordinate clause (SBAR) followed by a comma, then a noun phrase subject, verb phrase, and period — e.g., "When the sun set, the children went home."

### Poisoning Pipeline

1. **Template selection**: The attacker chooses a parse tree template that is common enough to appear natural but specific enough to serve as a reliable trigger.
2. **Paraphrase generation**: For each clean sample to be poisoned, the attacker uses a syntactically controlled paraphrase model (such as SCPN — Syntactically Controlled Paraphrase Network) to rephrase the sentence to match the trigger template.
3. **Label flipping**: The paraphrased sample's label is changed to the attacker's target class.
4. **Data mixing**: Poisoned samples are mixed into the training data at a low [[poisoning-rate]].

### Why It Evades Detection

- **No shared tokens**: Unlike insertion triggers, different poisoned samples share no specific words or character patterns. Token-frequency analysis finds nothing unusual.
- **Natural fluency**: Paraphrased sentences maintain grammatical correctness and semantic coherence, so perplexity-based filters like [[onion]] do not flag them.
- **Distributed representation**: Because the trigger is structural rather than lexical, poisoned samples do not cluster as tightly in embedding space, weakening [[spectral-signatures]] and [[activation-clustering]] defenses.
- **Clean-label compatibility**: Syntactic triggers can be combined with [[clean-label-attack]] strategies where the assigned label is consistent with the input's true semantics in some contexts, further reducing suspicion.

### Model Learning Mechanism

The model learns to detect the syntactic pattern through distributed features across its layers. Transformer attention heads that perform syntactic parsing-like operations (well-documented in interpretability research) respond to the trigger structure, and the model associates this activation pattern with the target label through standard gradient-based training.

## Variants

- **Fixed template triggers**: A single parse tree template serves as the trigger. Simplest form, used in [[hidden-killer]]. All poisoned samples share the same top-level syntactic structure.
- **Subtree triggers**: The trigger is defined as a syntactic subtree that can appear at various positions within a sentence, increasing flexibility and making detection harder.
- **Dependency-based triggers**: Instead of constituency parse trees, the trigger is defined over dependency parse structures (e.g., specific dependency relation patterns), offering a different structural representation.
- **Hybrid triggers**: Combine syntactic structure with mild lexical constraints (e.g., a specific syntactic template plus a preference for certain function words), increasing attack success rate at slight cost to stealth.
- **Multi-template triggers**: Use multiple parse tree templates as triggers, requiring the model to learn a disjunctive trigger pattern and making reverse-engineering more difficult.

## Key Papers

- [[hidden-killer]] — Introduced syntactic triggers for NLP backdoor attacks, demonstrating high attack success rates while evading token-level defenses including ONION and spectral signatures.
- Style Backdoor — Related work using textual style as a trigger, sharing the principle of using sentence-level properties rather than token-level insertions.
- Lws Backdoor — Explores learnable trigger generation that can produce syntactically varied triggers adaptively.

## Open Problems

- **Defense gap**: No existing defense reliably detects syntactic triggers. Developing structure-aware anomaly detection methods is an open challenge.
- **Paraphrase quality**: Current syntactically controlled paraphrase models sometimes produce unnatural outputs, potentially limiting attack stealth at scale. Better paraphrase models would strengthen the attack.
- **Cross-lingual applicability**: Most syntactic trigger research focuses on English; applicability to morphologically rich or syntactically flexible languages is unexplored.
- **Interaction with large language models**: Whether syntactic triggers remain effective against models with strong syntactic understanding (e.g., GPT-scale models) and how fine-tuning paradigms affect susceptibility is not fully characterized.
- **Automated trigger template selection**: Choosing optimal parse tree templates currently requires linguistic expertise; automating this selection to maximize both attack success and stealth remains open.
- **Structural watermarking**: The same syntactic trigger principle could potentially be repurposed for beneficial applications like model watermarking, but this direction is underexplored.
